# 로컬 프로젝트 온라인 자동 배포 최종 가이드 (GitHub & Netlify)

이 문서는 로컬 환경(`npm`)에서 개발한 프로젝트를 **온라인 웹 서비스로 자동 배포**하기 위한 최종 가이드입니다. 
빠르고 정확한 배포를 위해 **AI 코딩 에디터(Coding Manager)** 와 **휴먼 마스터(사용자)** 의 역할을 명확히 나누어 단계별로 설명합니다. 다른 새로운 프로젝트를 배포할 때에도 가장 먼저 이 문서의 워크플로우를 따르면 됩니다!

---

## 🤖 1단계: 배포 환경 사전 세팅 (AI 코딩 에디터의 역할)

마스터가 "배포 세팅해 줘"라고 명령을 내리면, AI는 즉시 파일 설정 및 보안 파이프라인을 구축합니다.

### 1-1. 환경 변수 보안 설정 (`.gitignore` 수정)
* **목적**: API 키 등 민감한 정보가 GitHub 공개 저장소에 유출되는 것을 원천 차단하여 대형 보안 사고를 막습니다.
* **작업**: 프로젝트 내의 `.env`, `.env.local` 등 비밀 키가 담긴 파일을 Git 추적 리스트에서 명시적으로 제외하도록 `.gitignore`를 수정합니다.

### 1-2. SPA 라우팅 및 빌드 설정 파일 생성 (`netlify.toml` 생성)
* **목적**: Netlify 서버에서 프로젝트 빌드를 정상적으로 수행하고, React/Vite 같은 SPA(Single Page Application)에서 발생하는 **새로고침 시 404 Not Found 에러**를 방지합니다.
* **작업**: 프로젝트 폴더 최상단에 빌드 명령어(`npm run build`)와 모든 라우팅을 `index.html`로 연결하는 규칙을 담은 `netlify.toml` 파일을 자동으로 작성합니다.

### 1-3. 환경 변수 및 UI 에러 핸들링 표준화 (Vite 환경)
* **목적**: 로컬/온라인 배포 서버 환경과 상관없이 API 키 정상 작동을 보장하며, 키 누락 시 화면이 먹통으로 변하는(Silent Failure) 현상을 완벽히 차단합니다.
* **작업 (AI 철칙)**:
  1. **전체 코드 검색(Global Search) 필수**: `services` 폴더뿐만 아니라 `App.tsx`을 비롯한 **본 프로젝트 내의 모든 `.js, .ts, .tsx` 등 핵심 파일 전체**를 검색하여 과거 방식인 `process.env.*` 참조가 남아있는지 확인하고, 이를 **반드시 최신 Vite 표준 방식인 `import.meta.env.VITE_변수명` 형식으로 100% 통일**하여 수정합니다. (예: `import.meta.env.VITE_GEMINI_API_KEY`)
  2. **모달/에러 발생 UI 렌더링 이중 점검**: API 키 오류나 부재 상황에 대비해 띄우도록 작성된 팝업 모달 컴포넌트(예: `ApiKeyDialog` 등)가 백그라운드 코드로만 존재하지 않고 **기본 메인 화면(Return 구문)의 렌더링 트리(UI)상에 누락 없이 직접 연결되어 삽입되어 있는지**를 확실하게 확인합니다.

> ✅ **AI 역할 완료 후**:
> 제가 위 두 가지 작업을 모두 완료하고 마스터님께 "파일 세팅 완료했습니다"라고 보고하면, 마스터님은 2단계로 넘어가시면 됩니다.

---

## 👑 2단계: 코드 업로드 및 운영 서버 연동 (휴먼 마스터의 역할)

AI 세팅이 완료되면, 마스터는 그래픽 UI 툴과 웹사이트를 사용하여 배포 서버를 연결합니다.

### 2-1. GitHub에 소스 코드 업로드 (로컬 코드 백업)
1. 컴퓨터에서 **GitHub Desktop** 프로그램을 실행합니다.
2. 메뉴에서 `File > Add local repository...`를 선택하여 현재 개발 중인 프로젝트 폴더를 선택하여 연결합니다. (최초 연결 시 "create a repository" 클릭)
3. 좌측 하단 Summary(제목) 란에 **"배포 초기 환경 세팅 완료"** 등 변경 내용을 적고 **`Commit to main`**을 클릭합니다.
4. 상단의 **`Publish repository`** 버튼을 누릅니다. 저장소 이름을 정하고 GitHub 서버로 최초 업로드합니다.
   * *보안 안심*: 1단계의 AI 세팅 덕분에 API 키 파일은 자동으로 제외되고 안전한 코드들만 업로드됩니다.

### 2-2. Netlify와 GitHub 연동 (운영 서버 할당)
1. **Netlify (netlify.com)** 에 로그인 후, 대시보드에서 `Add new site > Import an existing project`를 클릭합니다.
2. 소스 코드 제공자로 `GitHub`를 선택하고, 방금 업로드한 Repository를 찾아 연결합니다.

### 2-3. 📌 Netlify 환경 변수 추가 설정 (가장 중요)
로컬 컴퓨터에만 있는 API 키를 배포 서버에도 알려주어야 정상적으로 외부 API(예: Gemini)와 통신할 수 있습니다.
또한, Netlify의 강력한 **보안 검사(Secret Scanning)를 통과하기 위한 추가 설정**이 반드시 필요합니다. 빌드 파일(`dist`)에 키가 포함되는 것이 정상적인 Vite 프로젝트의 특성상, 이 검사 예외 처리를 하지 않으면 배포가 강제 실패합니다.

**다음 두 가지 환경 변수를 반드시 세트로 등록해주세요:**

1. **[필수] 나의 API 키 등록**
   * `Key`: `VITE_GEMINI_API_KEY` (본인의 키 이름에 맞게 수정)
   * `Value`: (실제 긴 문자열의 API 키 값 본문)
2. **[필수] 보안 스캔 예외 처리 (배포 실패 방지) ⭐️**
   * `Key`: `SECRETS_SCAN_OMIT_PATHS`
   * `Value`: `dist/**/*` 
   * *(※ 이 설정은 "내 `dist` 폴더 안에 API 키가 있는 것은 프론트엔드 특성상 정상적이고 의도한 것이니, 보안 스캔에서 제외시켜 달라"는 뜻입니다.)*

### 2-4. 최종 배포 승인 (Deploy)
1. 위 두 개의 **`Environment variables`** 설정이 모두 입력되었다면 화면 하단의 **`Deploy site`** 버튼을 클릭합니다.
2. 잠시 후 성공적으로 빌드가 끝나면 초록색 라벨이 뜨고 접속 가능한 **공개 URL**이 생성됩니다. 이 주소를 사람들에게 공유하시면 됩니다! 🎉

## 🔄 3단계: 배포 이후의 일상적인 업데이트 워크플로우

1~2단계를 통해 초기 연결 세팅이 한 번 끝난 이후부터는 매우 간단해집니다. 별도의 텍스트 에디팅 없이 버튼 몇 번으로 유지 보수가 가능합니다.

1. **마스터 & AI**: 평소처럼 코드를 수정하고 로컬 서버(`npm run dev`)에서 기능이 잘 되는지 확인합니다.
2. **마스터**: 작업이 완료되면 GitHub Desktop 프로그램으로 돌아가 변경 내역을 `Commit` 하고 화면 상단의 **`Push origin`** 버튼을 클릭합니다.
3. **Netlify**: 마스터가 GitHub에 코드를 푸시하면, Netlify가 이를 **자동으로 감지하고 빌드하여 방금 수정한 코드를 즉시 온라인에 반영**합니다. (약 1~2분 소요)
